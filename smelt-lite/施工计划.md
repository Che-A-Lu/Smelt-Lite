## 零、这是什么

**一个静态网页。** .card 格式的存在证明。

打开浏览器 → 拖文件进来 → 变成卡片 → 勾选 → 下载 .card。拖 .card 进来 → 看到里面有什么。三十秒。不需要安装、注册、服务器、AI 模型。

## 一、和旧 Smelt 的关系

| | 旧 Smelt（47 文件） | Smelt Lite（目标 ~10 文件） |
|---|---|---|
| 形态 | 浏览器工作台 | 静态网页 |
| 功能 | 画布 + AI 对话 + 工具 + 模式 + 导出 | 拖入 → 预览 → 导出 .card → 导入 .card |
| 存储 | OPFS 持久化 | 内存（关掉就没了，导出才持久化） |
| AI | 9 个模型、3 种协议 | 无 |
| 依赖 | React 18 + JSZip + OPFS | React + JSZip（OPFS 去掉） |

**旧 Smelt 代码保留在 `app/`，不动。Smelt Lite 是独立的新代码，放在 `smelt-lite/`。**

## 二、功能清单（最终交付物）

1. **拖入文件** → 自动识别类型，生成卡片预览（图片/文本/CSV/其他）
2. **拖入 JSON** → 如果符合 .card process.jsonl 格式，渲染为协作过程卡片
3. **拖入 .card** → 解包，展示里面有什么：文件列表、协作过程、签名信息
4. **卡片操作** → 查看详情、重命名、删除、勾选
5. **打包导出** → 勾选卡片 → 填名称/作者/标签 → 下载 .card（含签名）
6. **隐私扫描** → 导出前自动检测 API Key、邮箱、手机号，警告用户
7. **语言切换** → 中/英

### 不在这个版本里

- AI 对话、模型调用
- 画布无限缩放、拖拽物理
- 卡座、卡包、工作台
- 加密（AES）
- 联系人信任网络
- 沙箱脚本执行
- 模式卡（管线/团队/编排）
- OPFS 持久化

## 三、代码架构

```
smelt-lite/
├── index.html
├── package.json
├── tsconfig.json
├── vite.config.ts
├── src/
│   ├── main.tsx              ← React 入口
│   ├── App.tsx               ← 顶层：状态管理 + 面板路由
│   ├── foundation/
│   │   ├── types.ts          ← .card 格式类型 + 卡片类型
│   │   └── i18n.ts           ← 中英双语（精简到 ~30 key）
│   ├── features/
│   │   ├── import.ts         ← parseCardFile：解包 .card，读取 manifest + 文件列表
│   │   ├── export.ts         ← buildCardPackage：JSZip 打包 + downloadBlob + scanContent
│   │   ├── identity.ts       ← ECDSA P-256 密钥生成 + 签名（仅浏览器本地）
│   │   └── snapshot.ts       ← generateSnapshot：文本/图片/CSV 缩略图
│   ├── ui/
│   │   ├── DropZone.tsx      ← 拖放区：拖入文件/.card → 解析 → 生成卡片
│   │   ├── CardList.tsx      ← 卡片列表：网格布局 + 勾选 + 右键操作
│   │   ├── CardPreview.tsx   ← 单张卡片：缩略图 + 名称 + 类型角标 + 文件大小
│   │   ├── DetailView.tsx    ← 卡片详情浮层：图片全屏/文本滚动/CSV 表格
│   │   ├── ExportPanel.tsx   ← 导出面板：元信息表单 + 文件清单 + 隐私扫描结果
│   │   ├── ImportView.tsx    ← 导入预览：.card 内容展示 + 签名信息
│   │   └── Header.tsx        ← 顶栏：标题 + 语言切换 + 新建/清空按钮
│   └── styles.css            ← 全部样式
```

**注意：UI 组件库（是否用 Cladd/Tailwind）由角色 A（视觉设计师）决定。本计划只定义组件职责和接口，不定义视觉实现。**

## 四、从旧 Smelt 复用 vs 重写

### 可以直接复用的（改 import 路径即可）

| 文件 | 复用程度 | 需要修改 |
|---|---|---|
| `features/import/index.ts` | 90% | 去掉 zip bomb 检测的 Web Worker（简化为同步） |
| `features/export/index.ts` | 90% | 去掉 AES 加密部分，保留签名 |
| `features/identity/index.ts` | 95% | 几乎不动 |
| `features/snapshot.ts` | 95% | 几乎不动 |
| `foundation/i18n.ts` | 30% | 砍掉 70% 的 key，只保留通用 + 导出/导入相关 |

### 需要重写的

| 文件 | 原因 |
|---|---|
| `foundation/types.ts` | 旧版包含 Canvas/Dock/Pack/Workbench/InteractionConfig 等大量无关类型 |
| 全部 `ui/` | 旧版是画布 + 工作台，新版是简单的列表 + 面板 |
| `App.tsx` | 完全不同 |

### 不需要的

| 文件 | 原因 |
|---|---|
| `platform/storage.ts` | 不用 OPFS |
| `platform/settings.ts` | 不用 API Key |
| `features/ai/` | 不调 AI |
| `features/sandbox/` | 不跑脚本 |
| `features/mode/` | 不玩模式 |
| `features/templates/` | 不搞壳 |
| `features/tool-registry.ts` | 不用工具 |
| `features/identity/trust.ts` | 不做联系人网络 |
