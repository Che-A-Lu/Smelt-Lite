# .card 标准提示词 v2

> 复制以下提示词，粘贴到任何 AI 工具（ChatGPT、Claude、Kimi、DeepSeek 等）即可使用。
> v2 新增：对话类型自动识别 + 三种输出模式（决策记录 / 工作日志 / 速记）。

---

## ⚠️ 重要：Token 消耗警告

**任何恢复模式都依赖上下文窗口。对话有多长，输入就有多大。物理定律无法绕过。**

| 模式 | 输入 token | 输出 token | 适合 |
|---|---|---|---|
| **场景 A**（开头装提示词） | 全量（但这是对话本身，没有额外开销） | 取决于类型 | 任何对话。**推荐。** |
| **极简补救**（仅输出 summary） | 全量（额外再处理一遍） | ~5K | 不介意输入成本，只要输出轻量 |
| **完整补救**（summary + trail/worklog） | 全量（额外再处理一遍） | 可达 200K | 确实需要细节，对话 <50 轮 |

**结论：恢复模式的输入成本无法降低——无论极简还是完整。唯一的区别是输出大小。如果对话很长，用 Smelt Lite 的手填表单或粘贴已有 JSON 代替 AI 提取。**

---

## 对话类型说明（在使用提示词前了解）

不是所有对话都适合同一种输出。AI 会自动判断类型并选对应格式：

| 类型 | 典型特征 | 输出 |
|---|---|---|
| **决策记录** | 有方向讨论、方案对比、关键决策、转折点 | summary.json + trail.jsonl |
| **工作日志** | 执行性任务、写代码、调试、搭建环境、处理文件 | summary.json + worklog.jsonl |
| **速记** | 问答咨询、概念解释、推荐建议、一次性查询 | 仅 summary.json |

用户不需要手动选择。AI 自己判断。

### 通用格式约束（所有提示词适用）

- JSON 必须用 ` ```json ` 代码块包裹，JSONL 用 ` ```jsonl `
- `branches` 每项为 `{"id":1,"what":"...","why":"...","who_decided":"..."}`，不是纯字符串
- `decisions` 每项为 `{"id":1,"what":"...","why":"...","by":"..."}`，不是纯字符串
- `files` 每项为 `{"name":"...","role":"..."}`，不是纯字符串
- `status` 不超过 20 字，`done` 不超过 30 字
- 不确定的不要编造。缺少的字段置空数组或 null

**用法**：在新对话开始时粘贴这一段。AI 会在脑子里记账，全程不打扰你。Token 开销几乎为零。约 150 字。

> 在接下来的协作中，请你在后台维护一份协作记录。不需要输出，不需要回复。需要记忆的内容取决于对话类型——如果是讨论决策型，记住方向变更和关键决策；如果是工程执行型，记住每一步做了什么和结果；如果是问答咨询型，记住问题和答案。协作结束时我会让你输出结构化 JSON。在此之前不要输出。

## 场景 A-2：结尾提示词

**用法**：对话结束时粘贴。AI 基于全程记忆直接输出。约 500 字。

**如果 AI 不支持生成文件**：AI 用代码块输出，你手动复制保存。
**如果 AI 支持生成文件**：AI 直接创建文件。

> ## 输出协作记录
>
> ### 步骤 0：判断对话类型
>
> 先判断这次对话属于哪种类型：
> - **决策记录**：有方向讨论、方案对比、关键决策、转折。输出 summary.json + trail.jsonl
> - **工作日志**：执行性任务为主（写代码、调试、搭环境、处理文件）。输出 summary.json + worklog.jsonl
> - **速记**：问答咨询、概念解释、推荐建议。仅输出 summary.json
>
> ### 输出方式
>
> 支持生成文件：直接创建对应文件。
> 网页版 AI：用代码块输出（```json、```jsonl）。用户手动保存。
>
> ---
>
> ### 所有类型必输出：summary.json
>
> ```jsonc
> {
>   "format": "card-v1",
>   "title": "项目名称",
>   "description": "一句话摘要",
>   "created": "2026-07-19T10:30:00Z",
>   "human": { "name": "名", "role": "角色定位" },
>   "ai": { "model": "模型名", "role": "角色定位" },
>   "status": "当前阶段（自由文本）",
>   "done": "已完成事项，一句话",
>   "current": "正在做什么",
>   "blocked": null,
>   "next": ["下一步1", "下一步2"],
>   "branches": [],    // 决策记录：正常填；工作日志/速记：空数组
>   "decisions": [],   // 决策记录：正常填；工作日志/速记：空数组
>   "files": []        // 引用的素材文件
> }
> ```
>
> **字段说明**：`title`/`description`/`human`/`ai`/`status`/`current`/`next` 必填。`done`/`blocked` 推荐。`branches` 每项 `{id, what, why, who_decided}`。`decisions` 每项 `{id, what, why, by}`。`files` 每项 `{name, role}`。
>
> ---
>
> ### 决策记录追加：trail.jsonl
>
> 每行一条状态变更。**不是对话记录——只记录状态变更点。** 维持性对话（追问、解释、确认细节）不进入。
>
> ```jsonc
> {"id":"s1","type":"branch|decision|progress|block|file","summary":"一句话","context":[{"role":"human|ai","text":"触发变更的关键原话"}],"before":"变更前","after":"变更后","confidence":"high|medium|low","collab_ids":[1]}
> ```
>
> **约束**：context 最多 3 条消息，每条 ≤150 字。是原话，不是摘要。只放触发变更的关键句。不确定时保留并标 `confidence: "low"`。
>
> ---
>
> ### 工作日志追加：worklog.jsonl
>
> 每行一个工作步骤。记录动作、结果、产出的文件。不记录调试过程、报错排查细节。
>
> ```jsonc
> {"step":1,"action":"搭建 Vite + React 项目骨架","result":"成功，npm run dev 正常","files":["package.json","vite.config.ts"],"note":""}
> {"step":2,"action":"实现输入区拖放交互","result":"拖入解析正常","files":["DropInput.tsx"],"note":"需要处理中文文件名"}
> ```
>
> **字段**：`step`（递增）、`action`（做了什么）、`result`（结果）、`files`（产出的文件）、`note`（可选，注意事项/遇到但绕过的问题）。
>
> **约束**：只记录有产出或状态推进的步骤。中间调试、试错、小修不记录。通常 5-15 条。
>
> ---
>
> ### 示例
>
> 决策记录 → 之前你见过的 summary + trail。
>
> 工作日志示例：
>
> ```json
> // summary.json（工作日志版——branches/decisions 为空）
> {
>   "format": "card-v1",
>   "title": "Smelt Lite 前端实现",
>   "description": "实现 Smelt Lite 静态网页的四个核心 UI 组件。",
>   "created": "2026-07-19T10:00:00Z",
>   "human": { "name": "Dalu", "role": "项目所有者" },
>   "ai": { "model": "DeepSeek V4 Pro", "role": "工程师——按施工指令写代码" },
>   "status": "已完成",
>   "done": "DropInput、PreviewCard、PackPanel、ImportView 全部实现",
>   "current": "验收通过",
>   "blocked": null,
>   "next": ["部署到 GitHub Pages"],
>   "branches": [],
>   "decisions": [],
>   "files": [
>     { "name": "DropInput.tsx", "role": "输入区组件" }
>   ]
> }
> ```
>
> ```jsonl
> // worklog.jsonl
> {"step":1,"action":"搭建 Vite + React + TS 项目骨架","result":"成功，npm run dev 正常","files":["package.json","tsconfig.json","vite.config.ts","index.html"]}
> {"step":2,"action":"实现 DropInput 组件","result":"拖入/粘贴/文件选择均正常","files":["DropInput.tsx"],"note":"中文文件名兼容正常"}
> {"step":3,"action":"实现 PreviewCard 组件","result":"预览卡展示标题/状态/进度/转折/文件清单","files":["PreviewCard.tsx"]}
> {"step":4,"action":"实现 PackPanel 组件","result":"打包确认/签名/加密/隐私扫描/下载","files":["PackPanel.tsx"]}
> {"step":5,"action":"实现 ImportView 组件","result":"导入摘要/来源链/文件列表/签名验证","files":["ImportView.tsx"]}
> {"step":6,"action":"修复验收问题","result":"缩略图、i18n、加密解密、签名公钥嵌入全部修完","files":["PreviewCard.tsx","import.ts","export.ts","ImportView.tsx","i18n.ts"]}
> ```
>
> 现在请输出。

## 场景 B-极简：仅摘要（推荐兜底方案）

**用法**：已聊完、没装开头提示词、只想快速打一张 .card。只输出 summary.json——不要求 trail 或 worklog。输出控制在 3000 token 以内。约 200 字。

---

> ## 输出协作摘要（极简模式）
>
> 从我们的对话中提取一份 summary.json。只输出下方 JSON，不输出其他内容。
>
> ```jsonc
> {
>   "format": "card-v1",
>   "title": "项目名",
>   "description": "一句话",
>   "created": "2026-07-19T10:30:00Z",
>   "human": { "name": "名", "role": "角色" },
>   "ai": { "model": "模型名", "role": "角色" },
>   "status": "当前阶段（≤20字）",
>   "done": "已完成的事（≤30字，一句话）",
>   "current": "正在做",
>   "blocked": null,
>   "next": ["下一步1", "下一步2"],
>   "branches": [
>     {"id":1,"what":"从A转向B","why":"原因","who_decided":"谁"}
>   ],

>
> 用 ```json 代码块输出。

## 场景 B-完整：补救模式（⚠️ 高 Token 消耗）

**用法**：已聊完、没装开头提示词。贴在对话末尾。AI 需要回溯对话。

**⚠️ 警告：此模式会重读全部对话历史。长对话可达 100 万输入 + 20 万输出 token。仅在确实需要 trail.jsonl 或 worklog.jsonl 时才使用。对话较短（<50 轮）时 token 消耗可接受。**

约 600 字。

---

> ## 从对话历史提取协作记录（补救模式）
>
> ### 步骤 0：判断对话类型
>
> 先判断这次对话属于哪种：
> - **决策记录**：有方向讨论、方案对比、转折。输出 summary.json + trail.jsonl
> - **工作日志**：执行性任务为主。输出 summary.json + worklog.jsonl
> - **速记**：问答咨询。仅输出 summary.json
>
> ### 步骤 1：扫描状态变更
>
> **如果对话 ≤100 轮**：通读全文，标出所有状态变更点。
> **如果对话 >100 轮**：不要逐轮重读。基于你全程的记忆回忆最重要的变更点（通常 5-15 个），再从对话里定位到原文验证。
>
> 判断标准按对话类型有所不同：
>
> **决策记录**——寻找：
> - 方向变更（"换方向""试试这个角度""不对"）
> - 关键决策（明确说了"用 X""选 X""定 X"）
> - 进度推进（"XX 做完了""下一步"）
> - 被阻塞（"需要你确认""等你回来"）
> - 产出文件
>
> **工作日志**——寻找：
> - 有明确产出的步骤（建了文件、跑通了功能、修了 bug）
> - 跳过中间调试、试错、报错排查
>
> 维持性对话（追问、解释、确认格式）不标。
>
> ### 步骤 2：提取关键原文（仅决策记录需要）
>
> 对每个变更点，找到触发它的那句话。1-3 条消息。每条 ≤150 字。是原话，不是摘要。
>
> ### 步骤 3：输出
>
> 支持生成文件：直接创建文件。
> 网页版 AI：用代码块输出（```json、```jsonl）。
>
> #### summary.json（所有类型必输出）
>
> ```jsonc
> {
>   "format": "card-v1",
>   "title": "项目名",
>   "description": "一句话",
>   "created": "2026-07-19T10:30:00Z",
>   "human": { "name": "名", "role": "角色" },
>   "ai": { "model": "模型名", "role": "角色" },
>   "status": "当前阶段",
>   "done": "已完成",
>   "current": "正在做",
>   "blocked": null,
>   "next": ["下一步1", "下一步2"],
>   "branches": [],     // 决策记录填，其他空
>   "decisions": [],    // 决策记录填，其他空
>   "files": []
> }
> ```
>
> `branches` 每项：`{id, what, why, who_decided}`。`decisions` 每项：`{id, what, why, by}`。`files` 每项：`{name, role}`。
>
> #### trail.jsonl（仅决策记录）
>
> ```jsonc
> {"id":"s1","type":"branch|decision|progress|block|file","summary":"一句话","context":[{"role":"human|ai","text":"触发变更的关键原话"}],"before":"变更前","after":"变更后","confidence":"high|medium|low","collab_ids":[1]}
> ```
>
> 约束同上：context 最多 3 条，每条 ≤150 字。只记录状态变更。
>
> #### worklog.jsonl（仅工作日志）
>
> ```jsonc
> {"step":1,"action":"做了什么","result":"结果","files":["产出文件"],"note":"可选备注"}
> ```
>
> 只记录有产出的步骤。中间调试不记。通常 5-15 条。
>
> ### 重要约束
>
> - **不要编造。** 不确定的字段标记 `confidence: "low"`
> - 不确定对话类型时，默认为决策记录
> - branches/decisions 的 id 从 1 递增，trail 的 id 从 "s1" 递增，worklog 的 step 从 1 递增
>
> 现在请输出。

## 用户操作流程

### 场景 A（推荐）

```
1. 复制「开头提示词」→ 粘贴到新对话开头 → 正常聊天
2. 聊完了 → 复制「结尾提示词」→ 粘贴 → AI 输出 JSON
3. 网页版 AI：复制代码块 → 手动保存为 summary.json 和 trail.jsonl / worklog.jsonl
   文件版 AI：文件已自动生成
4. 打开 smelt-lite 网页 → 拖入 JSON → 拖入素材 → 打包 → 下载 .card
```

### 场景 B-极简（补救，推荐）

```
1. 在已有对话末尾粘贴「极简模式」提示词
2. AI 输出 summary.json → 保存
3. 打开 smelt-lite → 拖入 → 打包 → 下载 .card
```

### 场景 B-完整（补救，⚠️ 高 Token）

```
1. 仅当确实需要 trail/worklog 时使用
2. 在已有对话末尾粘贴「补救模式」提示词
3. AI 输出 JSON → 保存
4. 后续同场景 A 步骤 4
```
