# 多工作台 + 会话持久化 · 施工指令

## 现状

两个问题：

1. **同时只能开一个工作台。** `openWorkbench` 是 `string | null`。双击 B → A 的浮窗被关掉。
2. **关浮窗会话全丢。** 对话历史、上下文卡片、工具卡、模式卡——关了就没了。再打开，空的。

## 目标

- 双击工作台 A → 浮窗 A 打开。双击工作台 B → 浮窗 A 和 B 同时存在
- 关闭浮窗 → 会话自动保存。下次打开 → 全部恢复
- 多个工作台各自独立，互不干扰

## 一、多工作台支持

只改 `Canvas.tsx`：

```typescript
// 旧：const [openWorkbench, setOpenWorkbench] = useState<string | null>(null);
// 新：
const [openWorkbenches, setOpenWorkbenches] = useState<Set<string>>(new Set());
```

### 双击卡片 → 切换

```typescript
onDoubleClick={(id) => {
  if (card.isWorkbench) {
    setOpenWorkbenches((prev) => {
      const next = new Set(prev);
      if (next.has(id)) next.delete(id); else next.add(id);
      return next;
    });
  }
}}
```

### 渲染多个浮窗

```typescript
{[...openWorkbenches].map((id) => (
  <Workbench
    key={id}
    card={index.cards[id]}
    interaction={interaction}
    allCards={Object.values(index.cards)}
    registerZone={registerZone}
    unregisterZone={unregisterZone}
    onClose={() => setOpenWorkbenches((prev) => {
      const next = new Set(prev);
      next.delete(id);
      return next;
    })}
  />
))}
```

### Escape 键

```typescript
if (e.code === "Escape") setOpenWorkbenches(new Set());
```

## 二、会话数据结构

在 `ui/workbench/types.ts` 加：

```typescript
export interface WorkbenchSessionData {
  messages: WbMessage[];
  tempFiles: TempFile[];
  trayCards: string[];
  contextOrder: ContextItem[];
  toolItems: ToolItem[];
  modeCardId: string | null;
  localModel: string;
  autoMode: boolean;
  thinkingMode: string;
  contextOverrides: Record<string, { content: string; mode: "full" | "titleOnly"; note: string }>;
}
```

注：`contextOverrides` 从 Map 转 Record 再存（JSON 不支持 Map）。`modeCardData` 不直接存——从 `modeCardId` 对应的卡片内容重新解析。

## 三、存储函数

在 `platform/storage.ts` 加：

```typescript
const SESSIONS_DIR = "sessions";

export async function saveWorkbenchSession(
  workbenchCardId: string,
  data: WorkbenchSessionData,
): Promise<void> {
  const dir = await getRoot();
  const sessionsDir = await dir.getDirectoryHandle(SESSIONS_DIR, { create: true });
  const handle = await sessionsDir.getFileHandle(`${workbenchCardId}.json`, { create: true });
  const w = await handle.createWritable();
  await w.write(JSON.stringify(data));
  await w.close();
}

export async function loadWorkbenchSession(
  workbenchCardId: string,
): Promise<WorkbenchSessionData | null> {
  try {
    const dir = await getRoot();
    const sessionsDir = await dir.getDirectoryHandle(SESSIONS_DIR);
    const handle = await sessionsDir.getFileHandle(`${workbenchCardId}.json`);
    return JSON.parse(await (await handle.getFile()).text());
  } catch { return null; }
}

export async function deleteWorkbenchSession(workbenchCardId: string): Promise<void> {
  try {
    const dir = await getRoot();
    const sessionsDir = await dir.getDirectoryHandle(SESSIONS_DIR);
    await sessionsDir.removeEntry(`${workbenchCardId}.json`);
  } catch { /* 不存在就不删 */ }
}
```

## 四、Workbench 改动

### 4.1 挂载时加载会话

在 Workbench 初始化 `useEffect` 中（约第 60 行附近）：

```typescript
useEffect(() => {
  loadWorkbenchSession(card.id).then((saved) => {
    if (!saved) return;
    setMessages(saved.messages ?? []);
    setTempFiles(saved.tempFiles ?? []);
    setTrayCards(saved.trayCards ?? []);
    setContextOrder(saved.contextOrder ?? []);
    setToolItems(saved.toolItems ?? []);
    if (saved.modeCardId) setModeCardId(saved.modeCardId);
    if (saved.localModel) setLocalModel(saved.localModel);
    if (saved.autoMode !== undefined) setAutoMode(saved.autoMode);
    if (saved.thinkingMode) setThinkingMode(saved.thinkingMode);
    if (saved.contextOverrides) {
      for (const [ref, override] of Object.entries(saved.contextOverrides)) {
        contextOverrides.current.set(ref, override);
      }
    }
    // 恢复 modeCardData
    if (saved.modeCardId) {
      const card = allCards.find((c) => c.id === saved.modeCardId);
      if (card?.sphereId) {
        readSphereFiles(card.sphereId).then((files) => {
          try {
            const validated = validateModeCard(JSON.parse(files.map((f) => f.content).join("\n")));
            if (validated) { setModeCardData(validated); setModeType(validated.type); }
          } catch { /* */ }
        });
      }
    }
  });
}, []); // eslint-disable-line
```

### 4.2 自动保存（2 秒防抖）

```typescript
const saveTimer = useRef(0);
useEffect(() => {
  clearTimeout(saveTimer.current);
  saveTimer.current = setTimeout(() => {
    saveWorkbenchSession(card.id, {
      messages, tempFiles, trayCards, contextOrder, toolItems,
      modeCardId, localModel, autoMode, thinkingMode,
      contextOverrides: Object.fromEntries(contextOverrides.current),
    });
  }, 2000) as unknown as number;
  return () => clearTimeout(saveTimer.current);
}, [messages, tempFiles, trayCards, contextOrder, toolItems, modeCardId, localModel, autoMode, thinkingMode]);
```

### 4.3 关闭时最终保存

包装 `onClose`：

```typescript
const handleClose = useCallback(() => {
  clearTimeout(saveTimer.current);
  saveWorkbenchSession(card.id, {
    messages, tempFiles, trayCards, contextOrder, toolItems,
    modeCardId, localModel, autoMode, thinkingMode,
    contextOverrides: Object.fromEntries(contextOverrides.current),
  });
  onClose();
}, [messages, tempFiles, trayCards, contextOrder, toolItems, modeCardId, localModel, autoMode, thinkingMode, onClose]);
```

把所有调 `onClose` 的地方（底部按钮、归档）换成 `handleClose`。

### 4.4 清除时删除会话

`doClear` 末尾：

```typescript
contextOverrides.current.clear();
deleteWorkbenchSession(card.id);
```

归档**不删**——用户归档后可能还想回来。

## 六、验收

1. `npx tsc --noEmit` 零错误
2. 双击工作台 A → 浮窗 A 打开。双击工作台 B → 浮窗 B 也打开。A 和 B 并存
3. 在 A 里聊两句、加入上下文 → 关浮窗 → 再双击 A → 会话恢复
4. 关浏览器 → 重新打开空间 → 双击 A → 会话恢复
5. 点"清除工作台" → 再打开 → 空的
6. 点"归档" → 画布多一张记录卡 → 再打开 → 会话还在
7. 两个工作台各自的会话独立，不串数据
