---

## 一、整体定位

输入区从「单一拖放入口」升级为「三种来源 + 提示词引导」的复合面板。

设计要点：
- **三种来源平行**：用户一眼看到三个选项，互不嵌套。
- **当前任务优先**：手填表单强调「零 token」；粘贴代码诚实标注「高 token」。
- **提示词不离场**：用户在输入区卡住时，下方立刻给出口。

## 八、提示词入口

### 8.1 推荐方案：方案 B

把提示词入口放在**输入区底部**，而非 Header。

理由：
- Header 已有标题 + 身份按钮，再加一个按钮会显得拥挤。
- 用户在输入区犹豫「我没有 summary.json 怎么办」时，入口正好在视线附近。
- 文字链接比图标按钮更直接说明用途。

### 8.2 位置与样式

```
┌─────────────────────────────────────────────────────────────┐
│  （标签行）                                                  │
├─────────────────────────────────────────────────────────────┤
│  （内容区：拖放 / 表单 / 粘贴）                              │
├─────────────────────────────────────────────────────────────┤
│  不知道怎么生成？复制提示词 →                                │
└─────────────────────────────────────────────────────────────┘
```

- 高度：44px
- 背景：`--surface`（与输入区一致，因为是一体面板）
- 上边框：1px solid `--border`（与内容区分隔）
- 内边距：0 20px
- 文字："不知道怎么生成？复制提示词 →"
- 字号：13px / 500
- 颜色：`--accent`
- hover：颜色变深（`#2563EB`），右侧箭头右移 2px
- 点击区域：整行可点

### 8.3 备选：方案 A（Header）

如果 Dalu 坚持放在 Header，设计如下：

│ ◆ Smelt Lite              [提示词]  Dalu · a1b2..  ▼        │
└─────────────────────────────────────────────────────────────┘

- 「提示词」按钮：
  - 高度 32px
  - padding 0 12px
  - 背景 transparent
  - 边框 1px solid `--border`
  - 圆角 `--radius-sm`
  - 字号 13px / 500 / `--text-secondary`

## 十、状态与动效

### 10.1 标签切换

- 内容区淡入：opacity 0 → 1，150ms ease-out
- 底线滑动：transform translateX + width 变化，150ms ease-out（如用 indicator 实现）

### 10.2 表单验证

- 必填项未填时点「生成预览」：
  - 输入框边框变 `--error`
  - 输入框轻微晃动（translateX ±3px，200ms）
  - 标签下方出现 12px 红色提示："请填写标题"

### 10.3 浮层出现/消失

- 出现：opacity 0 → 1，transform scale(0.96) → scale(1)，150ms ease-out
- 消失：opacity 1 → 0，100ms ease-in
- 背景遮罩：`rgba(0,0,0,0.35)`，淡入 150ms

### 10.4 复制反馈

- 按钮文字切换：150ms
- 无额外动画，避免干扰

### 10.5 输入区底部链接 hover

- 箭头右移 2px，150ms ease-out
