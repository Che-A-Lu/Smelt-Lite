# 适配器层 · 施工指令

## 目标

让 DeepSeek 走原生 OpenAI 协议、Anthropic 走原生 Messages API、Gemini 走原生 Gemini API——而不是全部依赖兼容层。

每个适配器做三件事：请求翻译（平台格式→provider 格式）、响应解析（provider SSE→平台 StreamChunk）、Tool 映射（平台 ToolDef→provider tool 声明）。

## 不碰的东西

- `tool-registry.ts`（ToolDef 格式不变）
- `foundation/types.ts`（ChatMessage 格式不变）
- `platform/settings.ts`（API key 存储不变）
- Workbench.tsx 的 send 逻辑（只改模型选择 UI，不改发送流程）
- Canvas、CardView、Dock、PackView

---

## 一、新建 `features/ai/types.ts`

统一所有适配器的接口：

```typescript
import type { ChatMessage } from "../../foundation/types";
import type { ToolDef } from "../tool-registry";

// 适配器统一接口
export interface AdapterRequest {
  model: string;
  messages: ChatMessage[];
  tools?: ToolDef[];
}

export interface AdapterResponse {
  url: string;
  headers: Record<string, string>;
  body: string;  // JSON.stringify 后的请求体
}

export interface StreamChunk {
  content: string;
  thinking: string;
  done: boolean;
  error: string | null;
  toolCalls?: ToolCall[];
}

export interface ToolCall {
  id: string;
  type: "function";
  function: { name: string; arguments: string };
}

export interface Adapter {
  id: string;          // "openai" | "anthropic" | "gemini"
  label: string;       // 显示名
  buildRequest(req: AdapterRequest, apiKey: string): AdapterResponse;
  parseStream(response: Response): AsyncGenerator<StreamChunk>;
}
```

## 三、新建 `features/ai/adapters/anthropic.ts`

### buildRequest

```
url: https://api.anthropic.com/v1/messages
headers: x-api-key: ${apiKey}, anthropic-version: 2023-06-01, Content-Type: application/json
body:
  {
    model,
    max_tokens: 4096,       ← Anthropic 必填
    stream: true,
    system: <从 messages 里提取所有 system role 的消息拼接>,
    messages: <只保留 user / assistant role 的消息>,
    tools: <ToolDef → Anthropic tool 格式>,
  }
```

#### 消息转换

OpenAI 格式 → Anthropic 格式：

```
OpenAI: [{role: "system", content: "你是助手"}, {role: "user", content: "你好"}, {role: "assistant", content: "你好！"}, {role: "tool", content: "...", tool_call_id: "..."}]

       ↓

Anthropic system: "你是助手"
Anthropic messages: [
  {role: "user", content: "你好"},
  {role: "assistant", content: "你好！"},
  {role: "user", content: [{type: "tool_result", tool_use_id: "...", content: "..."}]}
]
```

规则：
- `role: "system"` → 提到顶层 `system` 字段（多个拼接换行）
- `role: "user"` → 原样
- `role: "assistant"` → 原样（如果包含 tool_calls，需转换为 content 数组中的 tool_use 块；但当前实现暂时跳过这个——历史 tool_call 只保留了 content 不保留 tool_calls 结构）
- `role: "tool"` → 转为 `role: "user"` + `content: [{type: "tool_result", tool_use_id, content}]`

#### Tool 格式转换

```
OpenAI: {type: "function", function: {name, description, parameters: {type: "object", properties, required}}}
       ↓
Anthropic: {name, description, input_schema: {type: "object", properties, required}}
```

#### tool_choice

```
OpenAI: "auto" → Anthropic: {type: "auto"}
```

### parseStream

SSE 事件类型映射到 StreamChunk：

| Anthropic SSE event | → StreamChunk |
|---------------------|--------------|
| `message_start` | 不产出，记录 message 元数据 |
| `content_block_start` (type: "text") | 不产出，记录 text block index |
| `content_block_start` (type: "tool_use") | 不产出，记录 tool_use id + name |
| `content_block_delta` (type: "text_delta") | `{content: delta.text, done: false}` |
| `content_block_delta` (type: "input_json_delta") | 累积 tool arguments，不产出 |
| `content_block_stop` | 不产出 |
| `message_delta` | 记录 stop_reason |
| `message_stop` | 吐出累积的 toolCalls，`{done: true}` |
| `ping` | 忽略 |

参考：[Anthropic 官方 streaming 文档](https://docs.anthropic.com/en/api/messages-streaming)

## 四、新建 `features/ai/adapters/gemini.ts`

### buildRequest

```
url: https://generativelanguage.googleapis.com/v1beta/models/${model}:streamGenerateContent?alt=sse&key=${apiKey}
headers: Content-Type: application/json
body:
  {
    contents: <messages 转换>,
    tools: <ToolDef → Gemini functionDeclarations 格式>,
    systemInstruction: <从 messages 里提取 system role>,
    generationConfig: { temperature: 0.7 }
  }
```

#### 消息转换

Gemini 的消息格式：

```
[{role: "user", parts: [{text: "你好"}]},
 {role: "model", parts: [{text: "你好！"}]},
 {role: "tool", parts: [{functionResponse: {name, response: {...}}}]}]
```

关键差异：
- OpenAI `assistant` → Gemini `model`
- OpenAI `tool` → Gemini `tool`（但 parts 结构不同）
- content 字符串 → `parts: [{text: "..."}]`
- system → `systemInstruction` 顶层字段

#### Tool 格式转换

```
OpenAI: {type: "function", function: {name, description, parameters}}
       ↓
Gemini: {functionDeclarations: [{name, description, parameters}]}
```

### parseStream

SSE 格式：

```
data: {"candidates":[{"content":{"parts":[{"text":"你好"}],"role":"model"},"finishReason":"STOP"}]}
```

解析规则：
- 读 SSE `data:` 行 → JSON.parse
- `candidates[0].content.parts` → 提取 text → `{content, done: false}`
- `candidates[0].content.parts` 中的 `functionCall` → 累积 toolCalls
- `finishReason` 出现 → `{done: true}`

## 五、改造 `features/ai/index.ts`

删除现有的 `resolveProvider` 和 `streamChat` 函数。改为：

```typescript
import { createOpenAIAdapter } from "./adapters/openai";
import { anthropicAdapter } from "./adapters/anthropic";
import { geminiAdapter } from "./adapters/gemini";
import type { Adapter, AdapterRequest } from "./types";

// OpenAI 兼容 provider → 用同一个适配器，只换 baseURL
const OPENAI_COMPAT = new Set(["deepseek", "openai", "kimi", "qwen", "zhipu", "groq", "custom"]);

function getAdapter(providerId: string, baseURL: string): Adapter {
  if (providerId === "anthropic") return anthropicAdapter;
  if (providerId === "google") return geminiAdapter;
  if (OPENAI_COMPAT.has(providerId)) return createOpenAIAdapter(baseURL);
  // fallback
  return createOpenAIAdapter(baseURL);
}

export async function* streamChat(
  modelId: string,
  messages: ChatMessage[],
  signal?: AbortSignal,
  tools?: ToolDef[],
): AsyncGenerator<StreamChunk> {
  // 1. 解析 provider + API key（逻辑不变）
  const resolved = await resolveProvider(modelId);
  if (!resolved) { yield error chunk; return; }

  // 2. 拿到适配器
  const adapter = getAdapter(resolved.config.id, resolved.config.baseURL);

  // 3. 构建请求
  const { url, headers, body } = adapter.buildRequest(
    { model: modelId, messages, tools },
    resolved.apiKey,
  );

  // 4. 调 fetch
  const response = await fetch(url, { method: "POST", headers, body, signal });
  if (!response.ok) { yield error chunk; return; }

  // 5. 用适配器解析流
  for await (const chunk of adapter.parseStream(response)) {
    yield chunk;
  }
}
```

核心变化：`streamChat` 从 170 行变成 ~40 行。所有 provider 特定的代码在适配器文件里。

## 六、Workbench 加模型选择器

在 Workbench 顶部模式切换旁，加一个模型下拉框。

数据来源：`BUILTIN_PROVIDERS` → 提取所有 provider 的 models → 扁平化列表。例如：

```
DeepSeek-Chat  |  DeepSeek-R1  |  Claude-Opus-4.7  |  Claude-Sonnet-4.6  |  GPT-4o  |  ...
```

每个选项显示 `provider-label / model-name`，如 `DeepSeek / deepseek-chat`。

行为：
- 默认值 = `getActiveModel()`（从设置读取）
- 用户选了新模型 → 覆盖当前工作台的 modelId
- 不影响全局设置
- 切换模型时不重置对话历史

最小 UI（一条 `<select>` + 一行状态文字）：

```
[自动·手动] | 模型: DeepSeek ▼ | Claude-Opus-4.7 丨GPT-4o 丨...
```
