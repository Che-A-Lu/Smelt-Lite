# 工具自动推荐 · 施工指令

## 目标

用户打字 → 按关键词自动把相关内置 tool 加入请求。用户不需要知道"该加哪个 tool"。

**不改模型选择。** 模型 = 用户手动选或链路卡写死。

## 改一个函数

`buildTools` 在 `features/tool-registry.ts` 第 153 行。

原来是：

```typescript
export function buildTools(toolItems: ToolItem[]): ToolDef[] {
  const tools: ToolDef[] = [...BUILTIN_TOOLS];
  if (toolItems.length > 0) { /* add script_run */ }
  return tools;
}
```

改成接受用户 prompt，按关键词补充：

```typescript
const KEYWORD_TOOLS: { keywords: string[]; toolName: string }[] = [
  { keywords: ["分析", "数据", "csv", "表", "查看", "读", "看看", "看一下", "读一下", "analyze", "data", "read", "look"], toolName: "card_read" },
  { keywords: ["列表", "有哪些", "所有的", "列出来", "list", "all"], toolName: "card_list" },
  { keywords: ["创建", "新建", "写", "生成", "产出", "create", "write", "generate", "new"], toolName: "card_create" },
  { keywords: ["改", "修改", "更新", "编辑", "update", "edit", "modify", "change"], toolName: "card_update" },
  { keywords: ["标签", "标记", "分类", "tag", "label", "category"], toolName: "card_tag" },
  { keywords: ["搜索", "查找", "找", "search", "find"], toolName: "card_search" },
];

export function buildTools(toolItems: ToolItem[], userPrompt?: string): ToolDef[] {
  const toolNames = new Set<string>();

  // 内置工具全加
  for (const t of BUILTIN_TOOLS) {
    toolNames.add(t.function.name);
  }

  // 关键词匹配 → 补充工具
  if (userPrompt) {
    const lower = userPrompt.toLowerCase();
    for (const kt of KEYWORD_TOOLS) {
      if (kt.keywords.some((kw) => lower.includes(kw))) {
        toolNames.add(kt.toolName);
      }
    }
  }

  // 工具区脚本 → script_run
  if (toolItems.length > 0) {
    toolNames.add("script_run");
  }

  // 按 toolNames 过滤 BUILTIN_TOOLS → 构建最终数组
  const tools: ToolDef[] = [];
  const added = new Set<string>();

  for (const bt of BUILTIN_TOOLS) {
    if (toolNames.has(bt.function.name)) {
      tools.push(bt);
      added.add(bt.function.name);
    }
  }

  // script_run 单独构建（需要注入脚本名称）
  if (toolNames.has("script_run") && toolItems.length > 0) {
    const scriptCards = toolItems.map((t) => t.label).join("、");
    tools.push({
      type: "function",
      function: {
        name: "script_run",
        description:
          `执行工具区中的脚本。可用脚本：${scriptCards}。` +
          "脚本在隔离沙箱中运行——无网络、无文件系统、无 DOM。",
        parameters: {
          type: "object",
          properties: {
            scriptName: { type: "string", description: "要运行的脚本名称" },
            input: { type: "string", description: "传给脚本的输入数据（可选）" },
          },
          required: ["scriptName"],
        },
      },
    });
  }

  return tools;
}
```

## 调用方加 prompt 参数

`buildTools` 的调用方在 Workbench 里两处：

**单模型模式**（第 411 行）：

```typescript
// 旧：const tools = buildTools(toolItems);
// 新：
const tools = buildTools(toolItems, text);
```

**编排模式**（`runOrchestrator` 调 `ctx.streamChat`）：
编排器不走用户 prompt 关键词——编排器是总指挥在调度。保持 `buildTools(toolItems)` 不加 prompt。

## 什么时候不加

- **编排模式**：总指挥有 `model_call` 足够
- **管线模式**：每步 instruction 已经写好了——那本身就是比关键词更精确的描述

结论：只在单模型模式和团队模式下加 prompt 关键词匹配。管线/编排不碰。
