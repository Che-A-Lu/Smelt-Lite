# 视觉设计稿：Smelt Lite

> 角色 A（视觉设计师），2026-07-19。
> 基于《Smelt Lite 施工计划》和《.card 格式规范 v2》。
> 输出：像素级布局、颜色、字体、间距、交互动效。

---

## 一、设计基调

### 1.1 气质

- **可信**：深蓝主色 + 卡片阴影，像一份正式的数字资产档案。
- **轻量**：单栏、居中、大量留白，不拥挤。
- **清晰**：信息分区明确，一眼能看出「这是什么」「到哪一步」「下一步是什么」。
- **克制**：不使用插画、渐变背景、圆角过大。科技感来自结构，不是装饰。

### 1.2 隐喻

把 Smelt Lite 想象成一台**放在桌面上的轻型塑封机**：

- 你把纸（summary.json）放上去 → 机器显示预览。
- 你追加附件 → 机器列出清单。
- 你按下按钮 → 机器盖戳（签名）、装袋（加密可选）、输出一个 .card。

所以视觉语言是：**入口大、操作区集中、反馈即时**。

## 十、交互动效

### 10.1 通用过渡

```css
transition: all 150ms ease-out;
```

### 10.2 拖放反馈

- 拖入文件进入窗口：输入区 150ms 内边框变蓝、背景变淡蓝
- 拖入合法文件（.json/.card）：输入区保持蓝色
- 拖入非法文件：边框变 `--error`，背景 `#FEF2F2`，200ms shake（左右 3px 晃动）

### 10.3 卡片出现

- 预览卡/打包面板/导入视图出现：
  - opacity 0 → 1
  - transform translateY(8px) → translateY(0)
  - 200ms ease-out

### 10.4 按钮反馈

- 悬停：背景变深，150ms
- 按下：scale(0.98)，100ms
- 禁用：opacity 0.5，cursor not-allowed

### 10.5 文件清单项

- 新项加入：从 opacity 0 淡入，高度从 0 展开到 44px，200ms ease-out
- 删除：opacity 1 → 0，高度 44px → 0，200ms ease-in

### 10.6 浮层

- 身份浮层：从上方 8px 处淡入，150ms
- 点击外部关闭
