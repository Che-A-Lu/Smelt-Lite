# 施工计划：Smelt Lite 输入区 v3

> 阿澈（角色 B），2026-07-19。
> 给角色 A（Kimi 2.7）做视觉设计。

---

## 背景

之前 Smelt Lite 只有一个入口：拖入/粘贴 JSON 文件。现在用户有三种来源来创建 summary：

1. 拖入已有的 summary.json 文件（保留）
2. 手填表单（新增）——不依赖 AI，零 token
3. 粘贴 AI 输出的代码块（新增）——AI 在别处生成，用户贴回来

三种方式平行，用户选哪个都行。同时页面上需要给用户提供提示词入口——让他知道怎么让 AI 生成。

---

## 一、输入区改造（DropInput.tsx）

### 1.1 三标签切换

当前默认的拖放区上方，新增一行标签。选中标签决定下方显示什么内容区。

```
┌──────────────────────────────────────┐
│  拖入文件  │  手填表单  │  粘贴代码   │
├──────────────────────────────────────┤
│                                      │
│         （内容区，随标签切换）          │
│                                      │
└──────────────────────────────────────┘
```

**标签样式**：横向排列，间距 24px。选中标签底部 2px 强调线，颜色 `--accent`。未选中标签 `--text-secondary`。字号 13px / 500。hover 变 `--text`。

**内容区**随标签切换，保留 150ms fade-in 过渡。

### 1.2 标签一：拖入文件（现有逻辑不变）

默认选中。就是现在的拖放区——200px 高，虚线边框，拖入/粘贴/选择文件。行为和样式不变。

### 1.3 标签二：手填表单

一张极简表单。4-5 个字段，纵向排列，间距 16px。

字段：

```
标题 *
┌──────────────────────────────────────┐
│ （必填）                              │
└──────────────────────────────────────┘

描述
┌──────────────────────────────────────┐
│ （一句话）                            │
└──────────────────────────────────────┘

当前阶段
┌──────────────────────────────────────┐
│                                      │
└──────────────────────────────────────┘

正在做
┌──────────────────────────────────────┐
│                                      │
└──────────────────────────────────────┘

下一步（一行一个）
┌──────────────────────────────────────┐
│                                      │
│                                      │
└──────────────────────────────────────┘

                [生成预览]
```

- 输入框：36px 高，1px `--border`，圆角 6px，padding 0 12px。聚焦 border `--accent`
- 标签：14px / 500 / `--text`，在输入框上方，间距 4px
- "生成预览"按钮：同现有"打包"按钮样式（`--primary` 背景，白色文字，36px 高），右对齐

表单右上角标注：

```
✨ 零 token 消耗
```

小字 12px / `--success`。

### 1.4 标签三：粘贴代码

内容区：

```
┌──────────────────────────────────────┐
│                                      │
│   粘贴 AI 输出的 JSON 代码块……        │
│                                      │
│                                      │
│                                      │
└──────────────────────────────────────┘
        [查看模板]    [解析并生成预览]
```

- textarea：最小 140px 高，`--border` 边框，圆角 10px，13px 等宽字体，padding 12px
- placeholder 13px / `--text-muted`
- "查看模板"按钮：次按钮样式（透明 + 1px border），左对齐
- "解析并生成预览"按钮：主按钮样式，右对齐
- 两按钮同行，左右分布

下方标注：

```
⚠️ 此方式需 AI 提取对话历史。对话越长，token 消耗越大。
```

小字 12px / `--warning`。

解析逻辑：模糊匹配——代码块里或裸 JSON 里找到 `"format": "card-v1"` 的对象即为 summary。找到 JSONL 数组即为 trail/worklog。格式不符时红色提示"未识别到有效 JSON"。

### 1.5 模板浮层

点「查看模板」弹出浮层：

```
┌─────────────────────────────────┐
│  summary.json 模板         [×]  │
├─────────────────────────────────┤
│                                 │
│  {                              │
│    "format": "card-v1",         │
│    "title": "项目名称",          │
│    "description": "一句话描述",   │
│    ...                          │
│  }                              │
│                                 │
│                         [复制]  │
└─────────────────────────────────┘
```

- 浮层宽度 480px，max-height 70vh，可滚动
- `--shadow-panel`，圆角 `--radius-lg`
- 代码区：等宽字体 13px，背景 `--bg`，圆角 `--radius-md`，padding 16px
- "复制"按钮：点击复制全文 → 按钮变"✅ 已复制" 2 秒

## 二、提示词入口

页面需要一个入口让用户知道怎么用 AI 生成 summary.json / trail.jsonl。

### 2.1 位置

两个候选，让视觉设计师选更合适的：

**方案 A**：Header 右侧，身份按钮旁边。加一个 "📋 提示词" 按钮。

**方案 B**：输入区下方，三个标签行右侧。加一个文字链接 "不知道怎么生成？复制提示词 →"。

### 2.2 提示词浮层

点击入口 → 浮层弹出：

```
┌──────────────────────────────────────────┐
│  .card 标准提示词                    [×]  │
├──────────────────────────────────────────┤
│                                          │
│  使用须知                                │
│  恢复模式会重读全部对话。对话越长，        │
│  token 消耗越大。推荐在对话开头就装提示词。 │
│                                          │
│  ┌─ 场景 A：对话前安装（推荐）────────┐  │
│  │ 在对话开始时粘贴以下提示词，AI 全程   │  │
│  │ 维护协作记录。结尾输出 JSON。        │  │
│  │                            [复制]   │  │
│  └────────────────────────────────────┘  │
│                                          │
│  ┌─ 场景 B：对话后提取（高 Token）────┐  │
│  │ 如果已经聊完了。可选极简（仅摘要）    │  │
│  │ 或完整（含过程记录）。              │  │
│  │                            [复制]   │  │
│  └────────────────────────────────────┘  │
│                                          │
└──────────────────────────────────────────┘
```

- 浮层宽度 520px，max-height 75vh，可滚动
- `--shadow-panel`，圆角 `--radius-lg`
- 每个提示词卡片：背景 `--bg`，边框 `--border`，圆角 `--radius-md`，内边距 16px
- 复制按钮：次按钮样式

卡片内显示提示词的**前 3-5 行预览** + 说明文字。复制按钮复制完整提示词内容。完整提示词存储在静态常量中（从 `提示词.md` 预置）。
