## 阶段二：上下文区 + 工具区 + File 区 + Tray 区

**目标**：四个区全部到位。用户能把画布卡片加入工作台上下文或工具集。

**要做什么**：

**上下文区（ContextZone.tsx）：**
- 显示已加入的卡片标签列表（卡片名 + 类型色块）
- `[+]` 按钮 → 弹出卡片选择器浮层（列出画布所有卡片，多选勾选）
- 每个标签可右键移除
- 拖拽画布卡片到上下文区区域 → 加入

**工具区（ToolZone.tsx）：**
- 同上结构，但标签颜色为紫色（#7c3aed）
- AI 工具声明从卡片 manifest.description 读取

**File 区（FilesZone.tsx）：**
- 监听 AI stream done 事件 → 追加临时文件
- 每个文件：色块 + 文件名 + 大小 + [建卡] 按钮
- 建卡 → card.create → 卡片出现在画布 + 加入 Tray 区

**Tray 区（TrayZone.tsx）：**
- 迷你缩略图（80×112px），水平排列
- 从工作台新建的卡、从 AI File 区建卡的卡、拖入工作台的外部卡 → 自动加入
- 可拖出回到画布

**拖拽到工作台**：
- 拖卡片到工作台浮窗 → 浮窗内各接收区高亮（上下文蓝色、工具紫色）
- 松手时判定指向哪个区 → 加入
- 不在任何区上 → 卡片归位

**验收**：四种方式加入卡片（拖到区、点+选、AI 产出、新建卡片），卡片在工作台内可见，拖出可回到画布。

## 阶段三：工具系统 + 内置工具集

**目标**：AI 能真正调用工具。内置工具集就位。

**要做什么**：
1. 定义 tool 声明格式（功能 + 限制 + 确认要求，见规范 4.8）
2. 建 `features/tool-registry.ts`——收集工作台当前可用的所有 tool 定义
3. 内置工具实现（Worker 沙箱里跑的逻辑）：

| tool | 逻辑 |
|------|------|
| `card.read` | 从 storage 读卡片内容 → 返回 |
| `card.list` | 列出工作台上下文区+工具区的卡片名+标签 |
| `card.search` | 按标签/名搜索画布卡片（只返名称+标签） |
| `card.create` | 创建新卡片到 AI File 区 |
| `card.update` | 修改一张卡的内容（限制：只能改工作台内的） |
| `card.tag` | 改卡片标签 |

4. 发送 AI 请求时，把工作台工具区的脚本卡 + 内置工具打包成 tools 数组，注入 API 请求
5. AI 返回 tool_call → Workbench 接收 → 调对应 tool → 结果还给 AI

**验收**：用户在工作台说"帮我看下这张数据表"→ AI 调 card.read → 拿到内容 → 回复分析结果。

## 阶段四：自动/手动模式 + 确认 UI

**目标**：用户控制 AI 的写入权限。

**要做什么**：
1. 设置面板加开关："工具执行模式：手动 / 自动"
2. 手动模式：AI 调写入类 tool → 弹出确认条 → [允许] [拒绝] [本次会话全部允许]
3. 自动模式：直接执行，状态条显示"AI 刚才调用了 card.update"
4. 只读类 tool 永远不弹确认
5. 模式可随时切换

**验收**：手动模式下 AI 想改卡片→弹确认→用户点允许才执行。切自动模式→不弹。

## 阶段五：工作台收尾

**目标**：归档、清除、工作台模板。

**要做什么**：
1. "归档全部"按钮 → 全部对话 + 上下文 + 工具 → 生成一张工作记录卡到画布
2. "清除工作台" → 删临时文件、归还卡片、关浮窗
3. 新建卡片按钮 → 小弹窗输入标题+选类型 → 直接在 Tray 区建卡
4. 工作台配置可导出为 .card（工作台模板）

**验收**：一次完整工作流——拖数据卡进上下文→发 AI→产出文件→建卡→归档。
