# 上下文注入与思考链 · 施工指令

## 目标

1. 工作台发送时，上下文区卡片内容**自动注入** system prompt（AI 不需主动调 card_read）
2. 用户可**预览和编辑**每张卡发给 AI 的内容（不影响原卡片）
3. 用户可右键**"仅发送标题"**，AI 只知卡片存在不知内容
4. AI 历史回复的思考链**默认发送最近一轮**，用户可三档控制
5. 用户可右键对话片段**"提取为卡片"**

---

## 一、上下文注入

### 1.1 改 Workbench `send()`——注入上下文卡片内容

在 `send()` 开头（构建 `apiMessages` 之前），加一段逻辑：

```typescript
// 收集上下文卡片内容 → 拼成 system message
const contextSystemMessages: ChatMessage[] = [];

for (const item of contextOrder) {
  if (item.type !== "card") continue;

  const card = allCards.find((c) => c.id === item.ref);
  if (!card?.sphereId) continue;

  // 检查是否有用户编辑过的覆盖版本
  const override = contextOverrides.current.get(item.ref);
  let content: string;
  let mode: "full" | "titleOnly";

  if (override) {
    content = override.content;
    mode = override.mode;
  } else {
    // 读卡片原始内容
    try {
      const files = await readSphereFiles(card.sphereId);
      content = files.map((f) => `[${f.name}]\n${f.content}`).join("\n\n");
      mode = "full";
    } catch {
      continue;
    }
  }

  if (mode === "titleOnly") {
    contextSystemMessages.push({
      role: "system",
      content: `用户提供了一张参考卡片：「${card.label}」` +
        (note ? `\n用户注释：${note}` : "") +
        `\n标签：${card.tags.join("、") || "无"}。内容未提供——仅在需要时通过 card_read 工具读取。`,
    });
  } else {
    contextSystemMessages.push({
      role: "system",
      content: `以下是用户提供的上下文卡片「${card.label}」的内容：` +
        (note ? `\n用户注释：${note}\n` : "\n") +
        `\n${content}`,
    });
  }
}

// 把所有上下文 system message 放在 apiMessages 最前面
apiMessages.unshift(...contextSystemMessages);
```

### 1.2 加 `contextOverrides` ref

每个工作台会话内维护一个 Map，存用户对每张上下文卡的编辑覆盖：

```typescript
const contextOverrides = useRef(new Map<string, {
  content: string;
  mode: "full" | "titleOnly";
  note: string;  // 用户注释：为什么放这张卡、AI 应该关注什么
}>());
```

这个 ref 在会话清除时清空。不影响原卡片数据。

## 二、上下文卡片预览与编辑

### 2.1 ContextZone 标签点击 → 预览浮层

点击上下文区的卡片标签 → 弹出"AI 可见内容"预览窗口：

```
┌─────────────────────────────────────┐
│ AI 将看到以下内容              [×]  │
│                                     │
│ 卡片：销售数据.csv                   │
│                                     │
│ 注释（可选）：                       │
│ ┌─────────────────────────────────┐ │
│ │ 这是Q3原始数据，重点看华东区      │ │
│ └─────────────────────────────────┘ │
│                                     │
│ 卡片内容：                           │
│ ┌─────────────────────────────────┐ │
│ │ 日期,销售额,区域                 │ │
│ │ 2024-01,12000,华东              │ │
│ │ 2024-02,13500,华东              │ │
│ │ ...（可编辑）                    │ │
│ └─────────────────────────────────┘ │
│                                     │
│ ○ 全内容发送                         │
│ ○ 仅发送标题（AI 通过 card_read 获取）│
│                                     │
│ [恢复原始内容]  [保存]               │
└─────────────────────────────────────┘
```

注释字段：用户在"注释（可选）"框里写一句话——这张卡为什么放进来、AI 应该关注什么。注释随卡片内容一起注入 system message。空着就不发。

行为：
- textarea 默认显示卡片原始内容，用户可编辑
- "全内容发送" / "仅发送标题" 二选一
- "恢复原始内容" → 重新从 OPFS 读取，覆盖编辑
- "保存" → 存入 `contextOverrides` ref，关闭浮层
- 关闭浮层但不保存 → 编辑丢弃

### 2.2 实现方式

ContextZone 新增一个 `onEditItem` 回调，传给 ZoneProps：

```typescript
// ZoneProps 加字段
onEditContextItem?: (ref: string) => void;
```

Workbench 收到回调后，弹出编辑浮层。浮层是一个独立组件 `ContextEditDialog`。

### 2.3 右键"仅发送标题"

ContextZone 右键菜单原有"删除"。改为：

```
删除
仅发送标题  ☐   ← 勾选态
编辑此卡发给AI的内容
```

勾选"仅发送标题" → `contextOverrides.current.set(ref, { content: originalContent, mode: "titleOnly" })`
取消勾选 → `contextOverrides.current.set(ref, { content: originalContent, mode: "full" })`

## 三、思考链发送

### 3.1 三档设置

在 Workbench 顶部（模式切换旁）加一个思考链开关：

```
［思考链：仅最近 ▼］
```

三档：

| 档位 | setting 值 | 行 为 |
|------|-----------|--------|
| 关闭 | `"off"` | 不发送任何思考链 |
| 仅最近一轮 | `"last"` | 只发上一条 AI 回复的思考链（默认） |
| 全部 | `"all"` | 所有历史 AI 回复的思考链都发 |

设置存在 `localStorage`：`"card-space-thinking-mode"`。默认 `"last"`。

### 3.2 怎么发送——不碰适配器

**不改适配器。** 思考链通过**文本拼接**发送——把思考内容追加到 AI 回复消息的 content 里。这比修改适配器简单，且跨所有 provider 通用。

在构建 `apiMessages` 时，对每条 `assistant` 消息：

```typescript
// 构建 API 消息时的思考链注入
for (const msg of apiMessages) {
  if (msg.role === "assistant" && msg.thinking) {
    const thinking = msg.thinking; // 存储在 ChatMessage 扩展字段里

    if (thinkingMode === "off") {
      // 不注入，只发 content
    } else if (thinkingMode === "last") {
      // 只对最近一条 assistant 消息注入
      if (isLastAssistantMessage) {
        msg.content = `[上一轮思考过程]\n${thinking}\n\n[回复]\n${msg.content}`;
      }
    } else if (thinkingMode === "all") {
      msg.content = `[思考过程]\n${thinking}\n\n[回复]\n${msg.content}`;
    }
  }
}
```

### 3.3 ChatMessage 类型扩展

`foundation/types.ts` 中 ChatMessage 加可选字段：

```typescript
export interface ChatMessage {
  role: "system" | "user" | "assistant" | "tool";
  content: string;
  tool_call_id?: string;
  tool_calls?: ToolCall[];
  thinking?: string;  // 新增：AI 的思考链（仅 assistant 消息）
}
```

### 3.4 Workbench 存思考链

当前 AI 回复存为 WbMessage，`thinkingTrace` 字段已有。构建 `apiMessages` 时映射过去：

```typescript
const apiMessages: ChatMessage[] = displayMessages.map((m) => ({
  role: m.role as "user" | "assistant",
  content: m.content,
  thinking: m.thinkingTrace || undefined,
}));
```

## 四、对话片段提取为卡片

### 4.1 右键 → "提取为卡片"

HistoryZone 里，每条消息的右键菜单加一项：

```
提取为卡片
```

点击后：
1. 取该消息的 `content`（如果勾选了多段，取所有勾选段）
2. 在工作台的 AI 文件区创建一张新卡片（名称 = "对话提取" + 时间）
3. 卡片内容 = 选取的消息文本（Markdown 格式，每条前加 `[用户]`/`[AI-模型名]` 标签）

实现：
- HistoryZone 的右键菜单加一行
- 回调 `onExtractCard(content: string)` → Workbench 调 `createCard` → 加入 FilesZone + TrayZone

## 六、验收

1. `npx tsc --noEmit` 零错误
2. 拖一张数据卡到上下文区 → 发送 prompt → AI 回复中引用了卡片数据（证明注入成功）
3. 点上下文标签 → 弹出预览浮层 → 编辑内容 → 保存 → 再次发送 → AI 看到的是编辑后的内容
4. 右键上下文标签 → "仅发送标题" → 发送 → AI 知道卡片存在但内容是空的
5. 思考链默认"仅最近" → 上一轮的思考链出现在 AI 收到的消息里
6. 切到"关闭" → 发送 → 思考链不出现
7. 右键对话消息 → "提取为卡片" → Files 区和 Tray 区出现新卡片
