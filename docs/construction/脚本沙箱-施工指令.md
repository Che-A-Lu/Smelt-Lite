## 现状

- `features/tool-registry.ts`：`executeTool` 中 `script_run` case 返回占位文字 `"尚未实现"`。**只差三行代码。**

## 改一个文件：`features/tool-registry.ts`

### 1. 加 import

```typescript
import { getSandbox } from "../sandbox/index";
import { readSphereFiles } from "../platform/storage";
```

### 2. 替换 `script_run` case（第 236-238 行）

旧：
```typescript
case "script_run": {
  return `脚本 "${args.scriptName}" 的执行尚未实现。`;
}
```

新：
```typescript
case "script_run": {
  // 按名称找工具区里的脚本卡
  const toolCardId = ctx.toolCardIds.find((id) => {
    const c = ctx.allCards.find((x) => x.id === id);
    return c?.label === args.scriptName || c?.label.toLowerCase() === args.scriptName?.toLowerCase();
  });
  if (!toolCardId) return `未找到脚本："${args.scriptName}"。请确认脚本卡已加入工具区。`;

  const card = ctx.allCards.find((x) => x.id === toolCardId);
  if (!card?.sphereId) return `脚本卡 "${args.scriptName}" 内容为空。`;

  // 读脚本内容
  let code = "";
  try {
    const files = await readSphereFiles(card.sphereId);
    code = files.map((f) => f.content).join("\n");
  } catch {
    return `无法读取脚本 "${args.scriptName}"。`;
  }

  // 沙箱执行
  const sandbox = getSandbox();
  const result = await sandbox.runScript(code);

  if (result.ok) {
    return result.output || "脚本执行完成（无输出）";
  }
  return `脚本执行错误：${result.error}`;
}
```

### 3. 不改 `ToolContext` 接口

`ToolContext` 已有 `toolCardIds: string[]` 和 `allCards: CardEntry[]`。不需要加字段。

### 4. 不改其他文件

`CardSandbox`、`executeTool` 调用方（Workbench）都不动。

## 验收

1. `npx tsc --noEmit` 零错误
2. 在工作台工具区加入一张脚本卡（如 `cleanup.py`：`return "cleaned 15 files";`）
3. 发送 prompt："帮我运行 cleanup.py"
4. AI 调 `script_run` → Worker 执行 → 返回 `"cleaned 15 files"`
5. 脚本有语法错误 → 返回 `"脚本执行错误：Unexpected token..."`
6. 脚本名称不存在 → 返回 `"未找到脚本：xxx"`
