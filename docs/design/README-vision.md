# 炼知 / Smelt

> 下一代人+AI 协作产出的开放格式。不是软件，是标准。空间是参考实现。
>
> 矿石进熔炉，金属出来。原始协作进空间，.card 出来。
> 炼 = smelt。知 = knowledge。

---

## 这是什么

一张 .card 是"一个人和 AI 的一次协作过程"的完整封装——过程、产物、元数据、编辑权、可继承性——打包成一个可拖拽、可分享、可编辑的 ZIP 文件。

格式开放，联邦式。任何空间都能打开任何 .card。不是平台，是语言。

## .card 格式（要点）

- **容器**：ZIP 包，后缀 `.card`
- **内部结构**：`manifest.json` + `process.jsonl` + `artifacts/` + `edits.json`
- **签名**：可选，ECDSA P-256，与内容解耦
- **加密**：可选，AES-256-GCM，解密后在任何空间可读
- **主题覆盖**：可选 `interaction` 字段，覆盖空间的交互参数（纯数字，不是代码）

## 空间是什么

空间是 .card 的解释器。极薄的浏览器应用。只做五件事：

1. **渲染卡片**——画布上的一切是卡片。对话框是卡片，右键菜单是卡片
2. **卡座**——卡片拖上去=通电，在 Worker 沙箱中运行。拖出=停用
3. **卡包**——收纳容器。可整体拖入卡座
4. **工作台**——会话卡。双击打开浮窗，在里面跟 AI 对话
5. **导入/导出**——.card 进出空间的门

空间不建功能。所有功能都是某个人做的某张卡在卡座上跑出来的。

## 关键设计决策

- **球不可见。** 球是 OPFS 里的数据实体，用户永远看不到它。用户的世界只有卡片
- **卡片不分类型。** 同一张脚本卡，放画布上是数据，拖进工作台是 AI 的工具，拖进卡座才通电跑
- **判定只在松手。** 拖拽经过卡座不激活。必须对准、松手才算
- **隐私编辑在导出阶段。** 不是加密，是编辑权
- **文件即应用。** .card 存在用户硬盘上，不锁在平台里

## 铁律

> 违反任何一条，阿澈有权拒绝合并。每条都可验证。

### 代码质量

1. **一个文件只做一件事。** 旧 Space.tsx 900 行塞了 50 个状态变量——那是屎山的根源。拆成独立文件：画布渲染、卡片拖拽、卡座、工作台浮窗，每个文件职责单一。衡量标准：能用一句话说清楚这个文件干什么。说不清 = 拆。单文件一般不超过 250 行，单组件不超过 5 个 `useState`
2. **不硬编码交互数字。** 所有动画时长、距离、比例、曲线从配置读取。改主题应该只改数字，不改代码
3. **功能通 ≠ 做完。** 必须对照交互规范逐条验证动画和物理感。`04-交互规范.md` 第十九章的检查清单是验收标准
4. **静态 import 优先。** 动态 import 只在必需的代码分割场景使用
5. **删死代码。** 未使用的 import、注释掉的逻辑、旧版 fallback——当场清理

### 架构强制

6. **四层单向引用。** foundation → platform → features → ui。反向 import = 编译错误处理
7. **只有 `platform/storage.ts` 碰 OPFS。** 任何其他文件直接调 OPFS API = 无效
8. **球不可渲染。** 球是 OPFS 数据实体。`SphereView.tsx` 不存在。任何 DOM 元素渲染球 = 无效
9. **卡片不分类型。** 不在代码里写 `if card.type === "function"` 之类的分支。行为由卡片在画布上的位置决定（在卡座里=通电，在工作台里=AI可见，都不在=静态）
10. **判定只在松手。** 拖拽过程中的邻近检测只做视觉反馈（高亮、磁吸），不做任何状态变更。松手那一刻才判定

### 体验铁律

11. **60fps 不可妥协。** 拖拽和滚动期间任何单帧超过 16ms = bug
12. **所有用户可见文字走 `t()`。** 硬编码中文字符串 = 无效
13. **每次改动后 `npx tsc --noEmit` 零新错误。** 改 `foundation/types.ts` 时手动检查所有引用方

### 安全

14. **卡座沙箱隔离。** 卡座上跑的脚本在 Web Worker 中，不碰 DOM、不碰网络、不碰 API key
15. **导入器不静默导入。** 签名失败、加密未解、可执行文件——必须展示并等用户确认

### 施工顺序

代码写在 `个人AI数字空间/v2/src/`。从零开始写 UI 层，复用 foundation + platform + features。

1. **画布**——无限平移+缩放+网格背景。只渲染，不管理数据
2. **卡片**——扑克牌比例 150×210px，带完整物理（延迟拾起、跟随、弹簧、惯性）
3. **卡包**——虚线框收纳容器，展开/折叠
4. **卡座**——磁吸底座，拖入=通电
5. **工作台**——会话卡，双击展开浮窗，四区面板
6. **面板和对话框**——设置、导出、导入、搜索
7. **连接 features**——画布调用 ai/、画布调用 import/export/、卡座调用 sandbox/
8. **存储持久化**——persist() 静默调用、状态条、配额监控

每做完一步，`npx tsc --noEmit` 零新错误，再进入下一步。
