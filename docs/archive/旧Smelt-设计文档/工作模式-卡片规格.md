# 工作模式卡 · 卡片规格

## UI 概念

工作台顶部，模型选择器旁边，加一个按钮：[工作模式 ▸]

点击 → 按钮下方展开一个凹槽区域。凹槽是一个 200×60px 的虚线框，提示文字"拖入工作模式卡"。凹槽旁边有一个可折叠的 `?` 帮助面板。

- 凹槽为空 → 工作台走单模型模式（默认行为，不变）
- 凹槽内有一张卡 → 工作台读卡的 `type`，切到对应模式
- 拖走卡片 → 回到单模型模式

**初学者永远不需要点这个按钮。高级用户点一下就能用。**

---

## 三种工作模式卡

### 1. 管线卡（`type: "pipeline"`）

串行步骤链。用户定义谁干什么、顺序是什么、中间是否暂停。

```json
{
  "type": "pipeline",
  "name": "数据分析 → 报告生成",
  "maxRounds": 10,
  "steps": [
    {
      "model": "deepseek-v4-pro",
      "instruction": "分析以下数据，提取关键结论和异常值。只输出结论，不要修饰。",
      "outputLabel": "分析结论",
      "contextMode": "all"
    },
    {
      "model": "claude-sonnet-4-6",
      "instruction": "根据上一轮的分析结论，撰写一份流畅、可读的分析报告。面向非技术读者。",
      "outputLabel": "分析报告",
      "contextMode": "last"
    },
    {
      "type": "pause",
      "label": "检查报告草稿，可编辑后继续"
    },
    {
      "model": "qwen-max",
      "instruction": "逐条检查报告中的数据结论是否与分析结论一致。列出不一致之处。没有问题就说'校验通过'。",
      "outputLabel": "合规检查",
      "contextMode": "last"
    }
  ]
}
```

**字段说明：**

| 字段 | 必须 | 说明 |
|------|:---:|------|
| `type` | ✅ | `"pipeline"` |
| `name` | ✅ | 管线名称（在凹槽中显示） |
| `maxRounds` | ❌ | 最大总轮次（默认 10）。防止无限循环 |
| `steps` | ✅ | 步骤数组，至少 2 步 |

**每步字段：**

| 字段 | 必须 | 说明 |
|------|:---:|------|
| `model` | ✅ | 模型 ID（从 BUILTIN_PROVIDERS 里选） |
| `instruction` | ✅ | 系统提示。描述这一步要做什么 |
| `outputLabel` | ❌ | 产出卡片的标签名 |
| `contextMode` | ❌ | `"all"` = 前面所有步骤的产出都给（默认），`"last"` = 只给上一步产出，`"none"` = 不给历史 |

**`pause` 步骤：** 没有 `model` 和 `instruction`，只有 `type: "pause"` + `label`。工作台在此暂停，弹出确认条显示 label，用户编辑中间卡片后点"继续"。

---

### 2. 团队卡（`type: "team"`）

多角色并行。同一个问题发给多个角色，各自独立回复。角色之间不对话。

```json
{
  "type": "team",
  "name": "论文评审团",
  "members": [
    {
      "model": "deepseek-v4-pro",
      "role": "数据质疑者",
      "persona": "你是数据分析专家，对每一个数字持有怀疑态度。找出方法论漏洞、样本偏差、统计误导。说'这里可能有问题'是你的职责。"
    },
    {
      "model": "claude-sonnet-4-6",
      "role": "业务分析师",
      "persona": "你是业务战略顾问。从行业视角解读数据，关注竞争态势、市场趋势、商业可行性。不要纠结技术细节。"
    },
    {
      "model": "qwen-max",
      "role": "保守派",
      "persona": "你是一个极度谨慎的风险管理者。对任何乐观结论都提出最悲观的解读。考虑边缘情况、黑天鹅事件、不可控变量。"
    }
  ]
}
```

**字段说明：**

| 字段 | 必须 | 说明 |
|------|:---:|------|
| `type` | ✅ | `"team"` |
| `name` | ✅ | 团队名称 |
| `members` | ✅ | 成员数组，至少 2 个 |

**每个成员：**

| 字段 | 必须 | 说明 |
|------|:---:|------|
| `model` | ✅ | 模型 ID |
| `role` | ✅ | 角色名（在对话历史区作为标签显示） |
| `persona` | ✅ | 角色人格描述（注入 system prompt） |

**行为：**
- 用户发送后 → 所有成员同时收到同一个 user prompt + 上下文区卡片内容
- 每个成员的回复在对话历史区以不同颜色标签展示（如 `质疑者·DeepSeek`、`分析师·Claude`）
- 成员之间不对话、不交叉引用
- 最后一条回复结束后，用户可以手动选择某条回复作为"最终答案"，或者人工综合

---

### 3. 编排卡（`type: "orchestrator"`）

一个总指挥模型动态调度其他模型。运行时决定调谁、给什么 prompt。

```json
{
  "type": "orchestrator",
  "name": "智能任务编排",
  "orchestratorModel": "claude-opus-4-7",
  "availableModels": ["deepseek-v4-pro", "claude-sonnet-4-6", "qwen-max"],
  "maxRounds": 8,
  "maxTokensPerRound": 8000,
  "systemPrompt": "你是一个任务编排器。用户给你一个目标。你决定：将目标拆解为几个子任务，每个子任务委托给最合适的模型执行。每次只委托一个子任务，等结果回来再决定下一步。不要自己完成子任务——你的职责是调度。"
}
```

**字段说明：**

| 字段 | 必须 | 说明 |
|------|:---:|------|
| `type` | ✅ | `"orchestrator"` |
| `name` | ✅ | 编排器名称 |
| `orchestratorModel` | ✅ | 总指挥模型 ID |
| `availableModels` | ✅ | 可调用的模型列表 |
| `maxRounds` | ❌ | 最大轮次（默认 8） |
| `maxTokensPerRound` | ❌ | 每轮最多多少 token（默认 8000） |
| `systemPrompt` | ❌ | 覆盖编排器的 system prompt |

**行为：**
- 总指挥模型收到一个特殊 tool：`model.call`
- `model.call` 参数：`model`（目标模型名）、`instruction`（要执行的任务）、`context`（传递的上下文）
- 每次 `model.call` 的产出生成一张临时卡片，出现在 AI 文件区
- 总指挥可以查看中间卡片内容（通过 `card.read` tool），决定下一步
- 达到 `maxRounds` 或总指挥输出最终结果时停止

## 凹槽的交互

凹槽是一个接收区，复用 Canvas 已有的 `registerZone` 机制。注册类型：`"mode"`。

1. 点击 [工作模式] 按钮 → 凹槽展开（150ms spring）
2. 凹槽为空 → 显示虚线框 + 提示文字 + `?` 按钮
3. 从画布拖卡片到凹槽 → 松手 → 检查卡片内容是否为合法的 mode JSON
4. 合法 → 卡片吸附到凹槽中（显示名称+类型标签）。工作台切到对应模式。
5. 不合法 → Toast："不是有效的工作模式卡"。卡片归位。
6. 从凹槽拖出卡片 → 卡片归位。工作台回到单模型模式。

## 第二步：`features/mode/pipeline.ts`

输入：`PipelineCard` + 用户的 prompt + 上下文卡片 + `streamChat` + `onStepOutput`

输出：逐步执行，每步结束后调用 `onStepOutput(stepIndex, cardId)` 将中间产物追加到 AI 文件区。

核心循环：

```typescript
async function runPipeline(
  card: PipelineCard,
  userPrompt: string,
  contextCards: CardEntry[],
  streamChat: (modelId: string, messages: ChatMessage[], signal?: AbortSignal, tools?: ToolDef[]) => AsyncGenerator<StreamChunk>,
  buildToolContext: () => ToolContext,
  onStepOutput: (label: string, content: string) => void,
  signal?: AbortSignal,
) {
  const maxRounds = card.maxRounds ?? 10;
  let round = 0;
  const stepOutputs: { label: string; content: string }[] = [];

  for (const step of card.steps) {
    if (signal?.aborted) break;
    if (round >= maxRounds) break;

    if (step.type === "pause") {
      // 暂停等用户确认。通过回调通知 UI 层。
      await waitForUserConfirm(step.label ?? "检查中间结果");
      continue;
    }

    round++;

    // 构建消息
    const messages: ChatMessage[] = [];
    messages.push({ role: "system", content: step.instruction! });

    // contextMode 决定把哪些历史产出放进去
    if (step.contextMode === "all") {
      for (const so of stepOutputs) {
        messages.push({ role: "user", content: `[${so.label}]\n${so.content}` });
      }
    } else if (step.contextMode !== "none") {
      // "last" 或默认：只传上一步
      const last = stepOutputs[stepOutputs.length - 1];
      if (last) messages.push({ role: "user", content: `[${last.label}]\n${last.content}` });
    }

    // 第一轮把用户 prompt 和上下文卡片加进去
    if (stepOutputs.length === 0) {
      if (userPrompt) messages.push({ role: "user", content: userPrompt });
      // 上下文卡片内容拼接
    }

    // 调模型
    let fullContent = "";
    const tools = buildToolContext ? buildTools(/* toolItems */) : undefined;
    for await (const chunk of streamChat(step.model!, messages, signal, tools)) {
      if (chunk.error) throw new Error(chunk.error);
      if (chunk.done) break;
      fullContent += chunk.content;
    }

    stepOutputs.push({ label: step.outputLabel ?? `步骤${round}`, content: fullContent });
    onStepOutput(step.outputLabel ?? `步骤${round}`, fullContent);
  }
}
```

`waitForUserConfirm` 用一个 Promise 实现——Workbench 展示确认条时 resolve。

## 第三步：`features/mode/team.ts`

输入：`TeamCard` + prompt + 上下文 + `streamChat`

输出：所有成员并行执行，每个回复追加为独立标签。

```typescript
async function runTeam(
  card: TeamCard,
  userPrompt: string,
  contextCards: CardEntry[],
  streamChat: ...,
  onMemberOutput: (member: TeamMember, content: string) => void,
  signal?: AbortSignal,
) {
  const tasks = card.members.map(async (member) => {
    const messages: ChatMessage[] = [
      { role: "system", content: member.persona },
    ];
    if (userPrompt) messages.push({ role: "user", content: userPrompt });
    // 拼接上下文卡片

    let fullContent = "";
    for await (const chunk of streamChat(member.model, messages, signal)) {
      if (chunk.error) throw new Error(chunk.error);
      if (chunk.done) break;
      fullContent += chunk.content;
    }
    onMemberOutput(member, fullContent);
    return { member, content: fullContent };
  });

  await Promise.all(tasks);  // 并行执行
}
```

## 第四步：`features/mode/orchestrator.ts`

输入：`OrchestratorCard` + prompt + 上下文 + `streamChat` + `buildToolContext`

输出：总指挥调度，中间产出卡片。

核心：给总指挥模型注入 `model.call` 这个特殊 tool。

```typescript
const MODEL_CALL_TOOL: ToolDef = {
  type: "function",
  function: {
    name: "model_call",
    description:
      "将一段子任务委托给指定的模型执行。" +
      "参数 model：目标模型名。" +
      "参数 instruction：要执行的任务描述。" +
      "可用模型：" + card.availableModels.join("、"),
    parameters: {
      type: "object",
      properties: {
        model: { type: "string", description: "目标模型名" },
        instruction: { type: "string", description: "要执行的子任务" },
      },
      required: ["model", "instruction"],
    },
  },
};

async function runOrchestrator(...) {
  const maxRounds = card.maxRounds ?? 8;
  let round = 0;

  const messages: ChatMessage[] = [
    { role: "system", content: card.systemPrompt ?? DEFAULT_ORCHESTRATOR_PROMPT },
    { role: "user", content: userPrompt },
  ];

  while (round < maxRounds) {
    round++;
    const tools = [MODEL_CALL_TOOL, ...buildTools(/* 当前 toolItems */)];
    let fullContent = "";
    let toolCalls: ToolCall[] = [];

    for await (const chunk of streamChat(card.orchestratorModel, messages, signal, tools)) {
      if (chunk.done) { toolCalls = chunk.toolCalls ?? []; break; }
      fullContent += chunk.content;
    }

    if (toolCalls.length === 0) {
      // 总指挥没有调 tool → 这是最终输出
      onFinalOutput(fullContent);
      return;
    }

    // 执行 model.call
    for (const tc of toolCalls) {
      if (tc.function.name !== "model_call") {
        // 其他 tool（card_read 等），正常执行
        const result = await executeTool(tc.function.name, parseArgs(tc.function.arguments), ctx);
        messages.push({ role: "tool", content: result, tool_call_id: tc.id });
        continue;
      }

      const args = JSON.parse(tc.function.arguments);
      // 调目标模型
      const subMessages: ChatMessage[] = [
        { role: "user", content: args.instruction },
      ];
      let subContent = "";
      for await (const chunk of streamChat(args.model, subMessages, signal)) {
        if (chunk.done) break;
        subContent += chunk.content;
      }
      onStepOutput(`${args.model}: ${args.instruction.slice(0, 50)}`, subContent);
      messages.push({ role: "tool", content: subContent, tool_call_id: tc.id });
    }
  }
}
```
