# 场景

## 背景推演过程

### 第一步：现状盘点

当前 AI agent 平台（Claude Code、扣子、GPT、Codex 等）本质上在做同一件事：**让 AI 从"回答者"变成"操作者"**。

三层拆解：
- 能力层：AI 能调用工具、读写文件、执行命令、操控浏览器
- 接口层：自然语言成为指令，替代鼠标键盘
- 集成层：把散落的工具串起来（终端+编辑器+Git / 大模型+插件+工作流）

但这些都未改变一个事实：**用户仍需主动打开它们，主动发出指令，主动做决策。** Agent 解决的是"执行"问题，不是"意图识别"问题。

这些平台的通用骨架只有三样：
1. 一个能理解意图的大模型
2. 一套可调用的工具
3. 一个调度层——模型判断何时用哪个工具

差异仅在工具种类、生态、UI 形态。

---

### 第二步：缺口识别

**当前核心缺口：人+AI 协作产出没有统一形态，无法被拥有、估值、流转。**

现有产出散落在聊天框里，具体表现为三个缺失：

1. **没有统一容器。** 代码有文件格式和仓库。但"人+AI 的产出"没有共识的打包方式——对话片段、模板、prompt、操作序列混在一起，无法独立存在。

2. **没有价值标注。** GitHub 有 star，论文有引用。但"我和 AI 花三小时搞出来的东西"——价值未被量化，甚至未被承认。只有产出者自己知道它的价值。

3. **没有流通机制。** 即使想给别人用，只能复制粘贴对话记录、录视频、写教程。每种方式都在丢失信息——别人拿到的只是截图，不是那个东西本身。

每个人都在各自平台上工作，但工作成果不能被积累、不能被交易、不能被继承。这个缺失不是技术问题——是还没有人把"人+AI 的产出"当做一个正经的东西来定义。

---

### 第三步：如何定义一个新东西

五步方法论：

1. **圈定边界**——说清楚"这个是，那个不是"。找到必需构成要素。
2. **找到最小不可分割单元**——完整协作产出拆到什么程度还能保持意义。黄金粒度：拆太小没价值，拆太大无法流转。
3. **定义内部结构**——至少容纳两层：
   - 过程层：怎么来的（意图、对话、修正、决策链路）
   - 结果层：最终产出是什么
   两层都不能少。只有过程没有结果，别人不知道怎么用。只有结果没有过程，退化成普通文件。
4. **定义它和人的关系**——三种关系：
   - 生产者：这个人和 AI 一起做出它
   - 使用者：另一个人直接拿来用
   - 改进者：拿来、理解过程、加入修改，产生新版本
   第三种关系是传统文件不支持但最需要的——它能**被延续**，不是一次性的。
5. **命名**——名字定义认知框架。需要一个词让人一听到就理解这不是文件、不是教程、不是聊天记录。

### 第四步：Dalu 的完整场景

> 当有一天，我打开电脑，打算工作。今天我要根据一份数据表做一个分析报告。我与 AI 共同完成。
>
> 我点开了一个目前不存在的软件或程序，里面什么都没有。我把数据表拖进了这个空间里。空间里出现了一张卡片，写着"XX数据表"，左上角一个绿色的角标，提示这是 Excel 文档。我的面前多了一个 API 调用模型选择的设置，包括 API key。我输入了。
>
> 我的面前有了一个工作台。我把刚刚的卡片拖进去了，工作台上有卡片了。底部有对话框。我说："帮我分析这个表，给我分析报告。"AI 调用了工具，开始读了。然后我原先的卡片旁边开始多很多个卡片——一个叫"分析报告"，其他都叫"AI临时文件"。分析报告现在是透明的，因为没写完。
>
> 最后做完了。任务完成。我点击"分析报告"这张卡片，角标提示我这是 MD 文档。我说"我要 word"。于是又过了一会，又多了一张卡，叫"word"。我点这张卡片，导出 word。我电脑的文件系统，干干净净，只出现了一个 word。其他临时文件都没有。
>
> 这时候，我回到工作台，所有卡片都还在。我点"导出为 .card"。于是出现了新的工作台——我的每一句 prompt，我的每个卡片，调用了什么工具，AI 回复了什么，都在上面。我可以增加或删减。我还可以把隐私的消息删除，我还可以加新的标注和解释，导入新文件，我还可以自定义这个卡的外观。
>
> 最后我点完成。一张 .card 出现了。
>
> 我把这个 .card 交给另一个人。他把 .card 拖进空间里。之前做的一切，就可以复现。他可以接手继续做下去。也可以模仿去做别的东西。
>
> 如果 .card 还可以是针对这个空间的程序——比如主题装饰、新功能补丁——让人自定义这个空间，无限自由度。.card 可以不是从这个空间里来的，它只是空间承认的一种文件，能够改写空间本身。

## 场景中析出的七层设计

### 第一层：空间 > 文件系统

工作台是一个沙盒。数据拖进去变成卡片，AI 生成的临时文件留在沙盒里，只有用户明确说"导出"的东西才落到真实文件系统。

"电脑干干净净，只出现了一个 word"——这是设计原则，不是顺便说的。当前电脑是文件系统第一，这个空间是工作台第一。

### 第二层：卡片是统一本体

Excel 是卡片，分析报告是卡片，AI 临时文件是卡片，.card 自己也是卡片。万物皆是卡片。

每种卡片有类型角标（绿标 Excel、MD、Word、.card），有不同的视觉状态（透明 = 生成中）。

这是一个统一的数据模型——不是"不同类型的文件"，而是"同一种东西的不同变体"。

### 第三层：过程可见，临时文件不污染产出

AI 在工作时产生一堆临时卡片，它们被标记、可见、可追溯，但导出时只带走需要的那一张。

这解决了当前 AI 工作台的核心矛盾：过程需要被看到，但结果需要干净。

### 第四层：.card 是工作台的快照 + 编辑器

点"导出为 .card"，打开的是一个编辑界面——包含每一条 prompt、AI 的每次工具调用、每个卡片、每条回复。

且它是可编辑的：可以删掉隐私消息、加标注、导入新文件、自定义外观。.card 不是过程的死记录，是过程的**可编辑版本**。

### 第五层：隐私是内置的，不是事后补救的

"删掉隐私消息"这个动作发生在导出阶段，不是使用阶段。分享之前，用户决定哪些东西别人能看到。这不是加密，是编辑权。

### 第六层：.card 是可继承的

另一个人把它拖进空间，"之前做的一切就可以复现，他可以接手继续做下去"。

这不是"看一遍学习"，是直接站在肩膀上继续。传统分享是"我给你看"，.card 是"我让你继续做"。

### 第七层：也可以不继承，而是模仿

"可以模仿去做别的东西"——.card 同时具有两种能力：作为模板被套用，和作为半成品被延续。两种不同的使用模式，都在一张 .card 里。

---

## 一句话定义

**一张 .card 是"一个人和 AI 的一次协作过程"的完整封装——过程、产物、元数据、编辑权、可继承性——打包成一个可拖拽、可分享、可编辑的单一文件。**

---

## 关键设计决策

### 平台锁定被刻意排除

场景中从未出现"登录 XX 平台""打开 XX 网页""需要 XX 会员"。.card 是本地文件，不依赖任何服务器。

这是一个**本地优先、文件即应用**的设计。

### 空名片 → 大教堂

如果一张空白的 .card 被拖进空间——它本身就是一段程序的入口，接受用户上传新文件、接受 prompt 指令、调用 AI 能力、生成新的卡片——人+AI 可以在里面建造出极度复杂的东西。

而所有复杂产物，都是从一张空 .card 开始的。"空"是生成力的起点。

### 空间无需被一个中央组织拥有

空间是一个 **.card 解释器**。如果一个群体共同维护他们的解释器，每个人都可以拥有无限自由的自定义空间——通过加载或修改各种 .card 来实现。软件本身极瘦，所有"功能"都用 .card 来定义。

---

## 权限模型

当 .card 能改写空间本身时，需要权限边界：

| 类型 | .card 能做的 | 它不能做的 |
|---|---|---|
| 数据级 | 读写工作台上的卡片内容 | 碰空间配置 |
| 外观级 | 改主题、字体、布局 | 碰数据 |
| 扩展级 | 注册新工具、新卡片类型 | 碰现有数据（除非用户明确授权） |

或更简单的方案：不搞分级，搞确认。拖入 .card 时，空间读它的声明，弹窗问用户。

---

## .card 的最终含义

如果主题、插件、补丁、工作流、分析报告——都是 .card——那么用户穿的衣服和用户产出的东西是**同一个格式**。

一个人把穿了主题、装了补丁、做出的分析报告打成一个 .card 发出去。收到的人看到的不只是报告，还包括这个人的整个工作环境——用什么颜色、装了什么辅助工具、用什么样的工作流。

这不是"我给你看我的产出"。
这是"我给你看我是怎么工作的，你也可以这样工作。"

**.card 不止能改写空间本身——.card 本身就是一种对空间的再定义。每个 .card 都是空间的一个变种、一套配置、一个 fork。**

空间不再是一个固定软件，而是一个能被无数 .card 重塑的基底。空间不拥有 .card。.card 不需要出生在空间里才能被空间识别。空间只是 .card 的一个解释器。.card 是**空间的语言**。

软件本身极瘦。所有的"功能"都用 .card 来定义——包括主题、插件、工作流、甚至未来出现的新卡片类型。.card 的上限是使用者想象力的上限。

这不是"一个软件"。这是对"人+AI 的产出"这个问题的完整回答——给它名字、形态、让它能被拥有、被估值、被流转。
