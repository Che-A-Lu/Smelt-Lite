# 格式主权

## 核心原则

**开放格式 > 封闭平台。**

一张 .card，无论在什么空间里被创建——浏览器版、大企业平台版、社区定制版——它在任何其他空间里都能被打开。不能打开的叫封闭格式，不叫 .card。

---

## 联邦式，不是中央式

每个空间是独立运行的 .card 解释器，不由任何单一公司拥有。

```
大企业平台 → 导出 → .card → 拖入 → 个人浏览器空间
个人浏览器空间 → 导出 → .card → 拖入 → 大企业平台
社区定制版 → 导出 → .card → 拖入 → 任何空间
```

**没有"平台锁"这种东西。** 平台可以附加自己的值（协作层、权限、审计），但它产出的 .card 剥离这些附加层之后，在最小解释器里依然能完整工作。

这是和当前所有 AI 平台的根本区别。ChatGPT 的对话导出来能在 Claude 里完整复现吗？不能。Notion 的文档导出来能在 Obsidian 里保持结构吗？只能部分。

.card 从一开始就把"在任何空间可读"写进格式定义，不是附加功能，是存在前提。

---

## 签名：可选的，且与格式解耦

每个人都可以给 .card 签名。

签名意味着什么：
- 这张 .card 是这个人签的
- 内容自签名后未被篡改
- 接收者可以选择只信任特定签名者的 .card

签名不改变 .card 的可读性。未签名的 .card 照样能打开。签了名的 .card 只是在元数据里多了一层可验证的来源信息。

签名的正确做法：
- 签名是 .card 格式的一个可选字段，不是必填
- 签名本身与内容分离——验证签名失败不影响读取内容
- 每个空间可以选择是否显示签名状态（"已签名·Dalu""未签名"）
- 任何人可以用任何密钥签名——不依赖中心化 CA

**签名回答的问题是"我可以信任来源吗"，不是"我能不能打开它"。**

---

## 加密：可选的，且不影响格式通读性

每个人都可以加密 .card。

加密有两种用法：
- **传输加密**：发送过程中加密，接收时解密。.card 本身存储为加密文件，打开需要密钥。
- **内容加密**：.card 内部某些字段（如 API key）加密，其他部分明文。空间读取时解密敏感字段，全部 .card 结构依然可读。

加密的正确做法：
- 加密是外层包装，不影响内部格式
- 解密后的 .card 和未加密的 .card 在结构上完全一致
- 加密密钥由用户持有，空间不代为保管
- 每个空间都能识别"这是一张加密的 .card"，即使打不开

**加密回答的问题是"谁有权读取它"，不是"这是什么格式"。**

---

## 本质：三件事各是各的

| 层 | 什么 | 谁决定 | 强制 |
|---|---|---|---|
| 格式可读性 | .card 在任何空间都能被打开和解释 | 格式规范 | 强制 |
| 签名（来源） | 谁签了这张 .card、内容是否被改过 | 创建者 | 可选 |
| 加密（访问） | 谁有权读取这张 .card | 创建者 | 可选 |

**格式可读性是地基。签名和加密是建在地基上由每个人自主选择的层。**

这三件事不混在一起，.card 就不会变成 DRM 文件。没有人能说"你必须用我的空间才能打开我的 .card"——因为 .card 不是加密格式，是开放格式。加密是可选的，且加密的 .card 解密后在任何空间都能打开，不是只在加密者的空间里。

---

## 大企业为什么杀不死这个

大企业可以做一个超强的空间——权限管理、实时协作、无限云存储、企业 SSO、审计追踪。它可以成为 .card 的最佳工作环境。

但它导出的 .card，和别人空间导出的 .card，是**同一个格式**。

这意味着：
1. 用户可以把企业平台里的 .card 拖到个人空间继续编辑
2. 企业平台不能把用户锁在里面——用户的 .card 数据是用户自己的
3. 企业必须靠平台的附加值留住用户，不能靠格式锁定

这和邮件的历史一模一样。Gmail、Outlook、个人邮箱——不同服务商，同一封邮件，同一套标准。不是因为 Google 和 Microsoft 道德高尚，而是因为 SMTP/MIME 是开放协议，谁不兼容谁就没法收发邮件。

.card 可以走同一条路。**你定的不是软件，是标准。**
